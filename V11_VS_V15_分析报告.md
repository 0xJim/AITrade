### B. V15新增的过滤条件(V11没有)

以下6项过滤是v12→v15逐步添加的，是信号从85→40笔减少的核心原因：

#### ❌ 过滤1: RSI≥65禁止做多 (v12新增)
```python
# v15 第1063行
if c["direction"] == "long" and tech.get("rsi", 50) >= 65:
    signals_filtered += 1; continue
```
**影响**: 过滤掉RSI在65-70之间的中强做多信号。虽然避免极端超买，但也排除了趋势初期RSI正在走强的信号。

#### ❌ 过滤2: ATR≥2%禁止做多 (v14新增)
```python
# v15 第1068行  
if c["direction"] == "long" and tech.get("atr_pct", 0) >= 0.02:
    signals_filtered += 1; continue
```
**影响**: 这是最严格的过滤！ATR≥2%的做多信号直接跳过。v10c数据中ATR 1.6%-2.7%的做多信号有多笔盈利，此规则一刀切删除太多。

#### ❌ 过滤3: ATR<0.7%禁止做空 (v15新增)
```python
# v15 第1102行
if c["direction"] == "short" and tech.get("atr_pct", 999) < 0.007:
    signals_filtered += 1; continue
```
**影响**: 低波动做空被完全禁止。但ORCAUSDT等低ATR做空在v10c中是盈利的。

#### ❌ 过滤4: 做空A级+ATR≥1.4%直接跳过 (v15新增)
```python
# v15 第1107行
if c["direction"] == "short" and c.get("strength") == "A" and tech.get("atr_pct", 0) >= 0.014:
    signals_filtered += 1; continue
```
**影响**: 仅保留S级做空中高ATR的，A级做空在ATR 1.4-2%区间全部被砍。v10c中SIRENUSDT(A级/ATR1.4%)等做空信号本来可以盈利。

#### ❌ 过滤5: 做多仓位强制减半 (v12新增)
```python
# v15 第1073行
long_factor = 0.5 if c["direction"] == "long" else 1.0
pos_usd = pos_usd * long_factor
```
**影响**: 做多仓位减半=每笔盈利减半。v11中做多仓位正常大小，盈利效应明显。

#### ❌ 过滤6: env_score=5仓位半价 (v14新增)
```python
# v15 第1083行
if c.get("env_score", 0) == 5:
    pos_usd = round(pos_usd * 0.5, 2)
```
**影响**: BTC方向不完全匹配时(env=5)仓位再砍半。这使得中质量信号盈利能力大降。

### C. V15信号排序变化

```python
# v10c/v11: 简单按strength和env_score排序
valid.sort(key=lambda x: ({...strength}, -env_score))

# v15: 按信号类型+做空优先排序
type_order = {"pump_short": 0, "extreme_pos_funding": 1, "coiling_breakout": 2}
valid.sort(key=sort_key)  # 做空优先于做多
```
**影响**: v15强制做空优先+做多次优先，导致做多信号被排在后面更容易被MAX_POSITIONS截断。

## 三、信号减少的量化分析

| 版本 | 信号发现 | 被过滤 | 实际交易 | 胜率 | PnL | 回撤 |
|------|---------|--------|---------|------|-----|------|
| v10c(V11基础) | 177 | 125 | 38 | 50% | +$576 | -6.3% |
| v12 | 177 | 125 | 39 | 51% | +$590 | -3.2% |
| v14 | 179 | 124 | 42 | 52% | +$607 | -4.1% |
| v15($1000) | 183 | 138 | 33 | 54.5% | +$115 | -4.2% |

**信号减少路径**: v11的85笔 → v15的33笔 = **减少52笔(61%信号被砍)**

### 信号损失分解(估算):

1. **ATR≥2%禁止做多**: ~15-20笔做多被砍(占最大)
2. **做空A级+ATR≥1.4%跳过**: ~8-10笔做空被砍
3. **ATR<0.7%禁止做空**: ~3-5笔做空被砍
4. **RSI≥65禁止做多**: ~5-8笔做多被砍
5. **做多仓位减半+env=5半价**: 不直接减少笔数，但大幅降低盈利
6. **做空优先排序**: 做多信号被截断~5-8笔

## 四、V11为何能达到69%胜率+85笔

### V11的核心优势:

1. **动态黑名单(+$568贡献)**:
   - 实盘中cron_scan自动统计近30天表现
   - 亏损>$50且胜率<40%的币种自动屏蔽
   - 这比静态黑名单精确得多，能避开短期表现差的币
   - v15只有静态黑名单，无法动态调整

2. **4h宽限期(止损失效率大降)**:
   - 入场后4h内不扫SL，避免假突破被扫
   - v10b数据: 持仓<12h的3笔全亏$246
   - 宽限期让信号有时间发展，胜率从56%→69%

3. **无ATR/RSI硬过滤(更多信号通过)**:
   - v10c/v11让所有通过env_score≥4的信号进入
   - ATR高也可以做多，RSI 65+也可以做多
   - 信号量更大=更多赚钱机会

4. **做多仓位正常大小**:
   - v11中做多仓位和其他一样大
   - v15做多减半后，即使盈利也赚得少，无法抵消亏损

5. **72h MAX_HOLD**:
   - v10c数据: 持仓>48h的7W/1L胜率88%赚$478
   - 延长持仓让更多信号到达TP

### 为什么v11有85笔而v10c只有38笔?

**最可能的原因**: v11的85笔来自**实盘运行6个月的累计**(2025年11月→2026年5月)，不是单次回测。实盘有:
- 动态黑名单不断更新
- 更多扫描周期(每小时/每4小时)
- 更多币种(不止回测中的50个)
- 而回测数据只是同一时间段的模拟，可能有差异

另一种可能: v11可能使用了v7plus的更宽松参数(MIN_ENV_SCORE=3, EXTREME_POS_FUNDING=0.08)，信号更多。

### 方案2: 添加动态黑名单到回测

```python
# 在回测循环中添加动态黑名单逻辑
# 每30天统计一次，亏损>$50且胜率<40%的币种屏蔽
if len(all_trades) > 0 and step_i % (30 * 24) == 0:
    symbol_stats = defaultdict(lambda: {"pnl": 0, "wins": 0, "total": 0})
    for t in all_trades[-200:]:  # 看最近200笔
        sym = t["symbol"]
        symbol_stats[sym]["pnl"] += t.get("pnl_usd", 0)
        symbol_stats[sym]["total"] += 1
        if t.get("pnl_usd", 0) > 0:
            symbol_stats[sym]["wins"] += 1
    dynamic_blacklist = set()
    for sym, stats in symbol_stats.items():
        wr = stats["wins"] / stats["total"] if stats["total"] > 0 else 0
        if stats["pnl"] < -50 and wr < 0.4:
            dynamic_blacklist.add(sym)
```

### 方案3: 混合策略(最稳健)

保留v15中**确实有效的过滤**，删除**过度过滤**:

| 过滤条件 | 保留/删除 | 理由 |
|---------|----------|------|
| RSI≥65禁止做多 | **放宽到75** | 避免极端超买即可 |
| ATR≥2%禁止做多 | **删除** | 一刀切太严，数据量不足 |
| ATR<0.7%禁止做空 | **改为仓位限制($700)** | 不直接跳过，而是减小仓位 |
| 做空A级+ATR≥1.4% | **删除** | 数据点太少(6笔)不具统计意义 |
| 做多仓位减半 | **改为0.8** | 略减但不过度 |
| env=5仓位半价 | **删除** | 过度压缩中质量信号盈利 |
| 做空优先排序 | **保留** | 数据支持做空胜率更高 |

**预期效果**: 信号量从33→55-65笔，胜率60-65%，PnL接近$800-900。

## 六、总结

v11的成功核心 = **宽松信号准入 + 动态黑名单 + 4h宽限期 + 72h持仓**

v15的问题 = **6层过滤层层叠加，每层单独看合理，但叠加后把61%的信号砍掉了**

**一句话: v15在"提高质量"的路上矫枉过正，需要回退到v11的"广撒网+动态淘汰"思路。**
