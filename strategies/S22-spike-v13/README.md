## 🎯 优化依据 (26笔历史订单分析)

| 优化项 | 数据支撑 | 效果 |
|--------|---------|------|
| 只做多 | 做空0胜率-9U，做多+254U | 消除亏损源 |
| EMA必须上升 | 顺趋势+254U vs 逆趋势-9U | 过滤逆势信号 |
| RSI≥50 | RSI<50全部亏损 | 过滤弱势信号 |
| 信号质量≥70 | 65-70区间11%胜率亏-111U | 杀掉死亡区间 |
| 持仓≤8h | >6h表现急剧下降 | 及时止盈/止损 |
| 质量≥80加仓×1.3 | 高质量信号更可靠 | 放大利润 |

## ⚙️ 关键参数 (config.py)

```python
SPIKE_LONG_ONLY = True              # 只做多
SPIKE_MIN_SIGNAL_QUALITY = 70       # 信号质量门槛 (65-70是死亡区间)
SPIKE_MIN_RSI = 50                  # RSI最低50
SPIKE_REQUIRE_EMA_UP = True         # EMA多头排列
SPIKE_MAX_HOLD_HOURS = 8            # 持仓时间上限
SPIKE_MIN_QUALITY_BOOST_MULT = 1.3  # 质量≥80加仓倍率
```

## 🔄 信号流程

```
15m K线异动(≥1%)
  → ATR≥0.5%
  → 只做多 (做空0胜率)
  → 1h EMA9>EMA21 (趋势确认)
  → RSI≥50 (动能确认)
  → 信号质量≥70 (综合评分)
  → 高质量≥80 加仓×1.3
  → 入场 (ATR×1.5止损, 最小3%)
  → 持仓>8h 强制平仓
```
